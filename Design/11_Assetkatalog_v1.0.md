# Lakeland Angler – Assetkatalog v1.0

## Zweck

Der Assetkatalog ist das zentrale Verzeichnis sämtlicher visueller Elemente des Buches.

Jedes Asset erhält eine eindeutige ID und kann später unabhängig aktualisiert oder ersetzt werden.

---

## Status

Jedes Asset besitzt einen Status:

- Geplant
- In Arbeit
- Fertig
- Im Layout verwendet

## Benennungskonvention

Alle erzeugten Dateien verwenden folgende Namensstruktur:

`Kategorie-ID_Beschreibung_Version`

Beispiele:

- `FISH-001_LargemouthBass_v1.png`
- `RIG-001_TexasRig_v1.png`
- `ICO-009_Alligator_v1.svg`
- `MAP-002_RetentionPond_v1.png`

Diese Struktur stellt sicher, dass Assets eindeutig referenziert, versioniert und später problemlos ersetzt werden können.
